# copilot-instructions.md

Pointer file for GitHub Copilot. Mirrors the content of `AGENTS.md` (which is the canonical source of truth for any AI harness in this workspace).

Why two files for the same rules? Because Copilot reads `copilot-instructions.md` from its own convention, not `AGENTS.md`. So this file exists as a harness-specific entry point. The body matches `AGENTS.md`; updates flow through one source.

Cross-harness setup: Claude Code (home), Copilot CLI (work).
